Policy per traffico dati

Si paga in base al traffico dati generato.

Con traffico dati si intende la somma delle dimensioni dei parametri di richiesta e di risposta dell'API, che comunque rimarrà limitato inferiormente ad 1 byte.

Questa policy è stata pensata per API caratterizzate da un passaggio di dati considerevoli, ad esempio la rielaborazione di immagini in alta risoluzione.

NBRQ: Numero byte dei parametri della richiesta all'API, calcolati secondo il protocollo SODEP
NBRS: Numero byte dei parametri della risposta dell'API, calcolati secondo il protocollo SODEP
VA: Valore attuale chiave API
VF: Valore finale chiave API
Vincoli: ( NBRQ + NBRS ) > 1
Formula: VF = VA - ( NBRQ + NBRS )
