Policy per tempo di utilizzo

Si paga in base al tempo di utilizzo dell'API.

Con tempo di utilizzo si intende il completo tempo di risposta dell'API, che comunque rimarrà limitato superiormente dalla SLA ed inferiormente ad 1 ms, anche se la risposta impiega 0 ms.

Questa policy è stata pensata per API che eseguono operazioni complesse che richiedono tanto sforzo computazionale, ma non ritornano necessariamente oggetti complessi, ad esempio calcoli tra matrici.

SLAMS: Millisecondi massimi garantiti dalla SLA
NMS: Numero millisecondi trascorsi tra l'invocazione della chiamata ed il ritorno della chiamata
VA: Valore attuale chiave API
VF: Valore finale chiave API
Vincoli: 1 < NMS < SLAMS
Formula: VF = VA - NMS
